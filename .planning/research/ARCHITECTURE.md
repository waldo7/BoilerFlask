## Data Flow

```
1. Request hits Flask вҶ’ Blueprint route matched
2. Route handler (routes.py) вҶ’ queries models вҶ’ renders template
3. Forms submit вҶ’ POST route вҶ’ validate (forms.py) вҶ’ model ops вҶ’ redirect
4. Auth: login вҶ’ Flask-Login sets session cookie вҶ’ @login_required gates
5. Templates: Jinja2 renders with {% extends "base.html" %} вҶ’ Bootstrap 5 CSS
```

## Build Order (dependencies)

```
Phase 1: Project Skeleton
  в”ңв”Җв”Җ config.py + extensions.py + app/__init__.py (factory)
  в”ңв”Җв”Җ base.html (layout + Bootstrap + mobile meta)
  в””в”Җв”Җ home.html (placeholder homepage)

Phase 2: Auth System
  в”ңв”Җв”Җ models/user.py (depends on Phase 1 extensions)
  в”ңв”Җв”Җ auth/ blueprint (depends on user model)
  в””в”Җв”Җ auth templates (depends on base.html)

Phase 3: Dashboard + Sidebar
  в”ңв”Җв”Җ dashboard route (depends on auth вҶ’ @login_required)
  в”ңв”Җв”Җ sidebar template (depends on base.html)
  в””в”Җв”Җ admin gating (depends on user.role)
```

## Key Architecture Decisions

| Decision | Rationale |
|----------|-----------|
| App factory over single app.py | Testable, multi-config, clean extension init |
| Blueprint per domain | Loose coupling; add features = add blueprints |
| Shared models/ directory | Single source of truth for DB schema |
| Extensions in separate file | Avoid circular imports (common Flask footgun) |
| config.py with classes | Env-based switching: Dev/Prod/Test |
