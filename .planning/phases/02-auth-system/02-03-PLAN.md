Create app/auth/forms.py per RESEARCH.md Pattern 3 with four FlaskForm subclasses.

Imports: from flask_wtf import FlaskForm; from wtforms import StringField, PasswordField, BooleanField; from wtforms.validators import DataRequired, Email, Length, EqualTo, ValidationError; import re; from app.models.user import User.

Class structure:
- LoginForm: fields email, password, remember. validate_email normalizes via strip().lower().
- RegistrationForm: fields email, password, confirm_password. Two custom validators. validate_email(self, field): email = field.data.strip().lower(); if User.query.filter_by(email=email).first(): raise ValidationError('Email already registered.'). validate_password(self, field): count = sum(bool(re.search(p, field.data)) for p in [r'[A-Z]', r'[a-z]', r'[0-9]', r'[^A-Za-z0-9]']); if count < 2: raise ValidationError('Password must contain at least 2 of: uppercase letter, lowercase letter, digit, or symbol.'). confirm_password uses EqualTo('password', message='Passwords must match.').
- ForgotPasswordForm: field email.
- ResetPasswordForm: fields password, confirm_password. Same custom validate_password as RegistrationForm. confirm_password uses EqualTo('password', message='Passwords must match.').

Use DataRequired() not InputRequired (DataRequired is the newer WTForms API). Use Email() from wtforms.validators (requires email-validator package — already installed in Plan 01).

DO NOT: route logic in forms. DO NOT: render_template calls. DO NOT: password hashing (that's in User model and routes). DO NOT: add role field to RegistrationForm (per D-09).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → Auth Forms | User input (email, password) crosses trust boundary via POST |
| Jinja Template → Browser | CSRF token rendered via hidden_tag() in form |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-02 | Elevation of Privilege | Auth forms CSRF | mitigate | All forms use FlaskForm with CSRFProtect; `{{ form.hidden_tag() }}` renders CSRF token in every form (per D-23/RESEARCH.md Pitfall 3) |
| T-02-04 | Elevation of Privilege | RegistrationForm | mitigate | No role selector on registration form; role defaults to USER server-side (per D-09) |
| T-02-12 | Information Disclosure | ForgotPasswordForm response | mitigate | Form does not reveal email existence; generic "check email" page rendered by route handler (per D-06) |
| T-02-16 | Information Disclosure | Password field | mitigate | Password show/hide toggle uses Bootstrap JS (no password visible in page source); input type="password" by default (per D-22) |
| T-02-17 | Tampering | Email field | mitigate | Email validator (wtforms Email + email-validator package) ensures RFC-compliant email format; strip().lower() normalization prevents duplicate casing tricks (per D-27/D-28) |
</threat_model>
