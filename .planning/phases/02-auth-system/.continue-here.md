# BLOCKING CONSTRAINTS — Read Before Anything Else

> These are not suggestions. Each constraint below was discovered through failure.
> Acknowledge each one explicitly before proceeding.

- [x] CONSTRAINT: Bootstrap CDN integrity hashes — auth/base.html had wrong integrity hashes that caused Bootstrap CSS to fail loading. Verified and fixed.
- [x] CONSTRAINT: Cross-Origin SRI Blocking — cross-origin links with an `integrity` attribute MUST have `crossorigin="anonymous"`. Missing it caused the browser to block Bootstrap CSS, leaving pages unstyled (Times New Roman). Verified and fixed.
- [x] CONSTRAINT: Flask-SQLAlchemy 3.x relative DB path — use `sqlite:///app.db` (no leading path). Relative paths under SA 3.x are prefixed with `instance_path`, creating `instance/app.db`. Verified and fixed.
- [x] CONSTRAINT: Trapped Session Notifications — Flash messages must be rendered and consumed immediately (e.g. on `/dashboard` and inside `base.html`). Failing to consume them on the dashboard causes banners like "Account created successfully" to linger in the session and leak onto the homepage on subsequent logout. Verified and fixed.

**Do not proceed until all boxes are checked.**

## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| Missing `crossorigin="anonymous"` | Missing on CDN link tags with an `integrity` attribute causes modern browsers to block style sheets completely, leaving pages unstyled | blocking | Always include `crossorigin="anonymous"` on cross-origin resources using SRI integrity hashes |
| Trapped flashed messages | If templates inheriting from base layout do not render flashed messages, notification alerts will linger in session state and print on unrelated pages | advisory | Include a global `{% with messages = get_flashed_messages() %}` rendering block inside the main column of `base.html` |
| SQLite relative URI doubling | Specifying `sqlite:///instance/app.db` under SA 3.x results in doubled pathing `instance/instance/app.db`. Use bare filename `sqlite:///app.db`. | blocking | Dev database SQLALCHEMY_DATABASE_URI must be relative (`sqlite:///app.db`) to reside natively inside `instance/app.db` |
