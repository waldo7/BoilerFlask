    if user is None:

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-05 | Spoofing | Flask-Login session | mitigate | login_manager.session_protection='basic' regenerates session ID on login (per D-15/RESEARCH.md §V3) |
| T-02-10 | Information Disclosure | Session cookies | mitigate | SESSION_COOKIE_HTTPONLY=True, SESSION_COOKIE_SAMESITE='Lax', REMEMBER_COOKIE_HTTPONLY=True in Config (per D-15) |
| T-02-14 | Elevation of Privilege | Deactivated user session | mitigate | user_loader returns None for is_active=False, terminating existing sessions (per D-25) |
| T-02-15 | Denial of Service | Auth POST endpoints | mitigate | Flask-Limiter default 10/min/IP on all auth POST endpoints (per D-29/D-30); memory:// storage for dev |
| T-02-08 | Tampering | SQLAlchemy queries | accept | SQLAlchemy ORM uses parameterized queries — no raw SQL in user_loader or queries; SQLi risk is low |
