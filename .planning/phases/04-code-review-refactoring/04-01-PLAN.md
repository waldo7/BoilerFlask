# Phase 4 Plan 1: Error Logging Refactoring

## Objective

Replace silent exception swallowing (`except Exception: pass`) in the dashboard and settings route handlers with `current_app.logger.error()` to ensure runtime failures are properly logged for debugging and observability.

## Tasks

<task>
<action>
Modify `app/auth/routes.py` to replace `except Exception:` blocks that silently pass with `current_app.logger.error()`.
Specifically:
1. Locate the `dashboard()` route handler and any other views (like `settings()` if it exists).
2. Ensure `current_app` is imported from `flask`.
3. In the `except Exception as e:` block, add `current_app.logger.error(f"Error in [view name]: {e}")`.
</action>
