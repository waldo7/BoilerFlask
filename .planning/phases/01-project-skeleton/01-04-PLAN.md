  <action>
    Create `run.py` at project root.

    Per RESEARCH.md §9 Component Specification:
    - Import `create_app` from `app`: `from app import create_app`
    - Create app at module level: `app = create_app()` — this makes `flask run` work because Flask auto-detects the `app` variable at module scope.
    - Add `if __name__ == '__main__':` guard: inside it, call `app.run(debug=True, host='0.0.0.0', port=5000)`. The guard allows `python run.py` for development while `flask run` bypasses it (uses the module-level `app` instead).

    **End-to-end verification:**
    After creating run.py, verify the complete application boots:
    1. Run `pip install -r requirements.txt` to install Flask + python-dotenv
    2. Run `python -c "from app import create_app; app = create_app(); print('✓ App created successfully')"` — confirms no import errors
    3. Run `python -c "from app import create_app; app = create_app(); client = app.test_client(); r = client.get('/'); assert r.status_code == 200; assert b'Flask app is running' in r.data; print('✓ Homepage renders OK')"` — confirms MAIN-01

    5. Run `python -m pytest tests/ -v` to run the full test suite
