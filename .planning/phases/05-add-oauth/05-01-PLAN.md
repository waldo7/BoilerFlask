# Phase 05: Add OAuth Integration

## Objective
Introduce OAuth authentication (Google & GitHub) using Flask-Dance alongside the existing email/password system, handle OAuth-only users securely, and provide UI elements for linking/settings.

## Must Haves
- `requirements.txt` includes `Flask-Dance[sqla]` and `blinker`.
- `config.py` contains variables for Google and GitHub OAuth secrets.
- `User` model allows `nullable=True` for `password_hash`.
- `OAuth` model tracks provider linking.
- `app/auth/oauth.py` configures blueprints and listens for `oauth_authorized` to link accounts and create new users.
- Login and Register pages display OAuth buttons.
- Settings page shows a "Set Password" form for users with no local password.
- Missing emails from providers fall back gracefully (e.g., flash error or redirect to manual entry).
